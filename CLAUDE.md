## What This Is

A Raycast extension that deploys services via `sdc` CLI. Provides a form UI for selecting a repo, service, Linear ticket, and deploy target, then orchestrates git operations + sdc invocations and copies PR links to clipboard.

## Commands

```bash
npm run dev       # Load extension in Raycast with hot-reload
npm run build     # Production build (ray build)
npm run lint      # Lint (ray lint)
npm run fix-lint  # Auto-fix lint issues
```

No test framework is configured.

## Architecture

Two Raycast commands:
- **deploy** (`src/deploy.tsx`) ‚Äî Form UI that orchestrates the full deploy flow
- **manage-overrides** (`src/manage-overrides.tsx`) ‚Äî List UI for managing service name overrides

Shared logic lives in `src/lib/`:
- `services.ts` ‚Äî Scans `{repo}/services/` to discover service names. Two-level walk: if a subdirectory has children, each child is a service; otherwise the subdirectory itself is. Falls back to repo name. Applies LocalStorage overrides.
- `sdc.ts` ‚Äî Wraps `sdc -d -s {service} -stage {stage} -ignore-tests -y -t {ticket}`. Parses PR URLs from stdout via regex. Also handles `git checkout main && git pull`.
- `linear.ts` ‚Äî `OAuthService.linear()` with `read` scope. Fetches up to 50 assigned non-completed tickets. Caches the LinearClient instance. Returns empty array on auth failure.
- `slack.ts` ‚Äî Extracts PR URLs from sandbox/live deploy results, joins with newlines.
- `storage.ts` ‚Äî LocalStorage CRUD for service name overrides. Key format: `override:{repoName}/{originalServiceName}`.

Types and stage constants are in `src/types.ts`.

## Key Behaviors

**Deploy targets and git strategy:**
- `unstable` / `staging` ‚Äî deploy from current branch, no git operations
- `sandbox` / `live` ‚Äî `git checkout main && git pull` before deploying
- `all` ‚Äî checkout main + pull, then deploy unstable ‚Üí staging ‚Üí sandbox ‚Üí live sequentially

**Service discovery ignore list:** `common` and `pkg` are always skipped. Additional directories configurable via the `ignoredDirectories` preference (comma-separated).

**sdc PATH:** Hardcoded to `/opt/homebrew/bin:/usr/local/bin:/usr/bin:/bin` because Raycast doesn't inherit shell PATH. The `sdcPath` preference allows setting an absolute path.

## Extension Preferences (package.json)

- `projectsDirectory` (required, directory) ‚Äî root dir with all repo clones
- `ignoredDirectories` (optional, textfield) ‚Äî extra dirs to skip in service discovery
- `sdcPath` (optional, textfield) ‚Äî absolute path to sdc binary
